# Internacionalização (i18n) — DocHub

O DocHub suporta múltiplos idiomas através de um sistema de i18n robusto configurável por variáveis de ambiente.

## Configuração Rápida

### 1. Variáveis de Ambiente

No seu arquivo `.env`:

```env
# Idioma principal (código de idioma: pt-br, en-us, es-es, etc)
LANGUAGE_CODE="pt-br"

# Idiomas suportados (separados por vírgula)
# Cria versões do site em cada idioma
SUPPORTED_LANGUAGES="pt-br,en-us"
```

### 2. Rodar Setup

```bash
cp .env.example .env
# Edite .env com seus idiomas

bash scripts/setup.sh
```

O script automaticamente:
- ✅ Configura o Hugo com os idiomas suportados
- ✅ Cria diretórios de conteúdo para cada idioma
- ✅ Atualiza `hugo.toml` com configuração de idiomas
- ✅ Carrega arquivos de tradução de `/i18n`

### 3. Estrutura de Conteúdo

```
content/
├── pt-br/
│   └── teams/
│       └── {team-id}/
│           ├── _index.md
│           ├── technical/
│           ├── product/
│           └── faq/
├── en-us/
│   └── teams/
│       └── {team-id}/
│           ├── _index.md (versão em inglês)
│           ├── technical/
│           ├── product/
│           └── faq/
└── es-es/
    └── teams/
        └── ...
```

## Uso

### Adicionar Novo Idioma

1. **Atualizar `.env`:**
   ```env
   LANGUAGE_CODE="pt-br"
   SUPPORTED_LANGUAGES="pt-br,en-us,es-es"
   ```

2. **Criar arquivo de tradução em `i18n/{lang}.yaml`:**
   ```yaml
   # i18n/es-es.yaml
   nav.home: "Inicio"
   nav.teams: "Equipos"
   # ... mais traduções
   ```

3. **Rodar update de configuração:**
   ```bash
   python3 scripts/manage-i18n.py
   ```

4. **Criar conteúdo:**
   ```bash
   mkdir -p content/es-es/teams
   # Copie seu conteúdo para content/es-es
   ```

### Gerar Documentação em Múltiplos Idiomas

Use o script `generate-docs.py` para gerar documentação em todos os idiomas suportados:

```bash
python3 scripts/generate-docs.py \
  --context context.json \
  --doc-types "technical,product,faq" \
  --project "api-payments" \
  --team "team-payments" \
  --output "content" \
  --languages "pt-br,en-us"
```

Isso criará documentação em:
- `content/pt-br/teams/team-payments/...`
- `content/en-us/teams/team-payments/...`

### Utilizar Traduções em Templates

No tema ou layout Handlebars/Go templates:

```html
<!-- Usando a função i18n do Hugo -->
<nav>
  <a href="{{ .Site.LanguagePrefix }}">{{ i18n "nav.home" }}</a>
  <a href="{{ .Site.LanguagePrefix }}/teams">{{ i18n "nav.teams" }}</a>
</nav>

<!-- Alternar entre idiomas -->
<select id="language-switcher">
  {{ range .Site.Languages }}
    <option value="{{ .LanguagePrefix }}" {{ if eq . $.Site.Language }}selected{{ end }}>
      {{ .LanguageName }}
    </option>
  {{ end }}
</select>
```

### Usando i18n em Scripts Python

No arquivo `generate-docs.py` ou scripts similares:

```python
from scripts.i18n_utils import load_i18n

i18n = load_i18n()

# Obter idioma principal
primary_lang = i18n.get_primary_language()  # "pt-br"

# Obter idiomas suportados
langs = i18n.get_supported_languages()  # ["pt-br", "en-us"]

# Traduzir uma chave
title = i18n.translate("doc.technical", language="en-us")  # "Technical"

# Caminho de conteúdo para um idioma
pt_content = i18n.get_content_path("pt-br")  # Path("content/pt-br")

# Criar frontmatter com informações de idioma
fm = i18n.create_language_frontmatter(
    base_frontmatter="---\ntitle: Test\n---",
    language="en-us",
    tags=["api", "backend"]
)
```

## Arquivo de Traduções (i18n/*.yaml)

### Estrutura

```yaml
# Navegação

# Interface

# Documentação

doc.faq: "FAQ"
```

### Idiomas Suportados

Por padrão, o DocHub inclui traduções para:

- **pt-br** — Português (Brasil)
- **en-us** — English (US)

Você pode adicionar novos idiomas criando arquivos em `i18n/{lang}.yaml` com os mesmos formatos.

### Linguagens Reconhecidas

- `pt-br` ou `pt-BR` — Português (Brasil)
- `pt` ou `pt-pt` — Português (Portugal)
- `en` ou `en-us` — English
- `en-gb` — English (GB)
- `es` ou `es-es` — Español
- `es-mx` — Español (México)
- `fr` — Français
- `de` — Deutsch
- `it` — Italiano
- `ja` ou `ja-jp` — 日本語
- `zh` ou `zh-cn` — 简体中文

## URLs Multilíngues

Quando múltiplos idiomas são configurados:

```
https://docs.suaempresa.com/       ← Idioma principal (pt-br)
https://docs.suaempresa.com/teams/ ← Português (Brasil)

https://docs.suaempresa.com/en-us/ ← Inglês (US)
https://docs.suaempresa.com/en-us/teams/

https://docs.suaempresa.com/es-es/ ← Espanhol (Espanha)
https://docs.suaempresa.com/es-es/teams/
```

**Nota:** O idioma principal NÃO usa prefixo na URL (fica na raiz).

## Script de Gerenciamento de i18n

Use `scripts/manage-i18n.py` para atualizar configuração de idiomas:

```bash
# Exporta variáveis de ambiente e roda script
export LANGUAGE_CODE="pt-br"
export SUPPORTED_LANGUAGES="pt-br,en-us,es-es"
python3 scripts/manage-i18n.py
```

O script:
- ✅ Lê variáveis de ambiente
- ✅ Atualiza `hugo.toml` com configuração de idiomas
- ✅ Remove seções de idioma antigas
- ✅ Verifica se arquivos de tradução existem
- ✅ Cria diretórios de conteúdo necessários

## Troubleshooting

### "Language files missing"

Se você vir aviso sobre arquivos de tradução faltando:

```
Warning: Missing i18n files for: en-us
```

Crie o arquivo:
```bash
touch i18n/en-us.yaml
```

E adicione suas traduções baseado em `i18n/pt-br.yaml`.

### Idioma não aparecendo no site

1. Verifique que `SUPPORTED_LANGUAGES` inclui o idioma
2. Verifique que `content/{lang}/teams/` existe
3. Rode `python3 scripts/manage-i18n.py` novamente
4. Limpe cache: `rm -rf resources/_gen/`

### URLs não funcionando

Certifique-se que:
1. `defaultLanguage` está correto em `hugo.toml`
2. `content/{lang}/` dirétórios existem
3. Arquivo `content/{lang}/_index.md` existe
